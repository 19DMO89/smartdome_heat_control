Smartdome Heat Control

🚀 Erste Schritte

Nach der Installation über HACS erscheint die Integration unter Einstellungen → Integrationen. Füge sie dort hinzu — danach öffnet sich dieses Panel automatisch.

  • Steuerung aktiv muss eingeschaltet sein, damit die Integration Thermostate steuert.
  • Lege über Raum hinzufügen deine Räume an oder erkenne vorhandene über Räume neu erkennen.
  • Klicke auf Konfiguration speichern um alle Änderungen zu übernehmen.
💡 Änderungen werden erst wirksam nachdem du gespeichert hast.

🌡️ Heizpumpensteuerung

Alle Felder sind optional — du kannst die Integration auch nur mit Heizkörperventilen ohne zentrale Steuerung nutzen.

  • Thermostat (Climate-Entity): Smartdome steuert einen Heizkreisregler (z.B. Lambdatronic, Fröling, Viessmann) per Solltemperatur. Der Boost-Delta-Wert hebt die Solltemperatur an, sobald ein Raum heizt, und senkt sie wieder wenn alle Räume warm sind.
  • Switch: Smartdome schaltet einen smarten Schalter (z.B. Shelly, Sonoff) direkt ein oder aus — kein Temperatursensor notwendig, kein Boost-Delta erforderlich.
  • Temperatursensor: Optional — wird nur zum Monitoring verwendet. Nützlich wenn die Climate-Entity keinen guten internen Sensor hat.
💡 Wenn du mehrere Heizkreise hast (z.B. Erdgeschoss / Obergeschoss), kannst du unter Heizkreise jedem Kreis eine eigene Heizpumpensteuerung zuweisen.

⚙️ Heizmodi

  • Comfort: Ventile bleiben länger offen. Weniger Temperaturschwankungen, Komfort im Vordergrund. Mehr Energieverbrauch möglich.
  • Balanced: Empfohlener Standard. Gute Balance aus Komfort und Energieeffizienz.
  • Energy: Ventile schließen früher. Reduziert Überhitzen, Energiesparen im Vordergrund. Leichte Temperaturschwankungen möglich.
  • Adaptive: Lernt das Verhalten jedes Raums und stoppt die Heizung früh genug, damit die Restwärme den Raum exakt auf Solltemperatur bringt. Braucht einige Heizvorgänge zum Einlernen.

🔁 Heizkreise

Heizkreise sind nur notwendig wenn dein Gebäude mehrere physisch getrennte Heizkreise hat (z.B. Fußbodenheizung Erdgeschoss + Heizkörper Obergeschoss), die jeweils über eine eigene Heizpumpensteuerung gesteuert werden.

  • Erstelle einen Heizkreis über Heizkreis hinzufügen und weise ihm eine Heizpumpensteuerung zu.
  • Weise Räume diesem Kreis über das Feld Heizkreis im Raum zu.
  • Räume ohne Kreis verwenden weiterhin die globale Heizpumpensteuerung.
💡 Wenn du nur einen Heizkreis hast, brauchst du hier nichts konfigurieren.

🌡️ Außentemperatur-Abschaltung

Wenn ein Außentemperatursensor konfiguriert ist, kann die Heizung automatisch unterdrückt werden, sobald die Außentemperatur einen Schwellenwert erreicht oder überschreitet.

  • Außentemperatursensor: Beliebiger Sensor oder Climate-Entity mit Temperaturattribut.
  • Abschaltung aktiv: Muss aktiviert sein damit die Funktion greift.
  • Schwellenwert: –20 °C bis +30 °C (Standard: 15 °C). Wird die Außentemperatur erreicht oder überschritten, wird die Heizung nicht eingeschaltet.
💡 Raumthermostate verbleiben auf ihrem normalen Idle-Sollwert — kein Frostschutzmodus. Die Funktion reagiert in Echtzeit auf Sensorzustandsänderungen.

🏠 Räume

  • Thermostat: Der Heizkörperthermostat dieses Raums. Wird direkt gesteuert.
  • Temperatursensor: Optionaler externer Sensor (z.B. Wandsensor). Wird für die Regelung verwendet wenn gesetzt.
  • Fensterkontakte: Beliebig viele Fenster- oder Türkontakte. Sobald einer offen meldet, pausiert die Heizung für diesen Raum.
  • Zieltemperatur Tag / Nacht: Gewünschte Raumtemperatur je Tageszeit.
  • Away-Temperatur: Temperatur wenn der Nicht-Zuhause-Modus aktiv ist.
  • Wochenschema: Individuelle Zeitsteuerung für diesen Raum — überschreibt die globalen Zeiten.
  • Kalibrierungsoffset: Korrigiert einen fixen Messfehler des Raumthermostats (–5 bis +5 °C, in 0,5 °C Schritten). Nützlich wenn das Thermostat z.B. systematisch 1 °C zu hoch regelt.

🧠 Adaptive Heizung

Im Adaptive-Modus lernt die Integration wie stark jeder Raum nach dem Abschalten noch nachschwingt (Overshoot). Die Integration schaltet die Heizung früher ab, damit die Restwärme den Raum exakt auf Solltemperatur bringt.

Das Lernen passiert in drei Zeitbuckets — je nachdem wie lange der Raum heizt:

  • Kurz (< 15 min): Wenig Restwärme, kleiner Overshoot-Wert.
  • Mittel (15–45 min): Typischer Heizbetrieb.
  • Lang (> 45 min): Starkes Aufheizen, mehr Restwärme im System.

Die gelernten Werte werden gespeichert und bleiben nach einem Neustart erhalten. Nach 5–10 Heizvorgängen je Bucket ist die Regelung spürbar genauer.

💡 Die aktuell gelernten Werte werden in der Raumkarte unter Adaptive Overshoot angezeigt.

💡 Tipps & häufige Fragen

  • Steuerung aktiv aus: Die Integration schreibt keinerlei Sollwerte auf Thermostate — sie greift komplett nicht ein.
  • climate-Entity als Sensor: Trage eine climate-Entity als Temperatursensor ein — die Integration liest automatisch das Attribut current_temperature aus.
  • Mehrere Fensterkontakte: Pro Raum können beliebig viele Sensoren hinzugefügt werden. Es reicht wenn einer offen ist um die Heizung zu pausieren.
  • Räume verschwinden nach Reload: Nutze Konfiguration speichern bevor du Neu laden klickst — sonst gehen ungespeicherte Änderungen verloren.
  • HACS-Update erscheint nicht: Stelle sicher dass die Versionsnummer in manifest.json mit dem GitHub-Release-Tag übereinstimmt.

🚀 Getting Started

After installing via HACS, the integration appears under Settings → Integrations. Add it there — this panel will open automatically.

  • Control enabled must be switched on for the integration to control thermostats.
  • Add rooms using Add room or discover existing areas via Reload rooms.
  • Click Save configuration to apply all changes.
💡 Changes only take effect after you save.

🌡️ Main Thermostat & Temperature Sensor

Both fields are optional — you can use the integration with only per-room thermostats.

  • Main thermostat: Controls all rooms that have no dedicated thermostat. It is set to the target temperature as soon as at least one such room wants to heat.
  • Temperature sensor: Used for measuring only — the integration never writes to it. Useful when the thermostat's built-in sensor is inaccurate.

A climate entity (e.g. a radiator thermostat) can also be used as a sensor — the integration then reads the current_temperature attribute automatically.

💡 If you have multiple heating circuits (e.g. ground floor / upper floor), assign a dedicated main thermostat to each circuit under Heating circuits.

⚙️ Heating Modes

  • Comfort: Valves stay open longer. Less temperature fluctuation, comfort first. Higher energy use possible.
  • Balanced: Recommended default. Good balance of comfort and efficiency.
  • Energy: Valves close earlier. Reduces overheating, energy saving first. Slight temperature swings possible.
  • Adaptive: Learns each room's behavior and stops heating early enough for residual heat to bring the room exactly to the target temperature. Needs a few heating cycles to calibrate.

🔁 Heating Circuits

Heating circuits are only needed if your building has multiple physically separate circuits (e.g. underfloor heating ground floor + radiators upper floor), each controlled by its own main thermostat or pump.

  • Create a circuit via Add heating circuit and assign a main thermostat to it.
  • Assign rooms to the circuit using the Heating circuit field in the room card.
  • Rooms without a circuit use the global main thermostat.
💡 If you have only one heating circuit, you don't need to configure anything here.

🌡️ Outdoor Temperature Cutoff

When an outdoor temperature sensor is configured, heating can be automatically suppressed once the outdoor temperature reaches or exceeds a threshold.

  • Outdoor temperature sensor: Any sensor or climate entity with a temperature attribute.
  • Cutoff enabled: Must be active for the feature to take effect.
  • Threshold: –20 °C to +30 °C (default: 15 °C). Once the outdoor temperature reaches or exceeds this value, heating is suppressed.
💡 Room thermostats stay at their normal idle setpoint — no frost protection mode. The feature responds in real time to sensor state changes.

🏠 Rooms

  • Thermostat: The radiator thermostat for this room. Controlled directly by the integration.
  • Temperature sensor: Optional external sensor (e.g. wall sensor). Used for control logic when set.
  • Window sensors: Any number of window or door sensors. If any one reports open, heating is paused for that room.
  • Day / Night target temperature: Desired room temperature for each time of day.
  • Away temperature: Temperature when away mode is active.
  • Weekly schedule: Individual time control for this room — overrides the global times.
  • Calibration offset: Corrects a fixed measurement error in the room thermostat (–5 to +5 °C, in 0.5 °C steps). Useful if the thermostat consistently regulates 1 °C too high, for example.

🧠 Adaptive Heating

In Adaptive mode the integration learns how much each room overshoots after the heating stops. It switches off the heating early enough for residual heat to bring the room exactly to the target temperature.

Learning happens in three duration buckets — depending on how long the room has been heating:

  • Short (< 15 min): Little residual heat, small overshoot value.
  • Medium (15–45 min): Typical heating cycle.
  • Long (> 45 min): Heavy heating, more residual heat in the system.

Learned values are persisted and survive restarts. After 5–10 heating cycles per bucket the control becomes noticeably more accurate.

💡 Current learned values are shown in the room card under Adaptive Overshoot.

💡 Tips & FAQ

  • Control disabled: The integration writes no setpoints to any thermostat — it does not intervene at all.
  • Climate entity as sensor: Enter a climate entity as the temperature sensor — the integration automatically reads its current_temperature attribute.
  • Multiple window sensors: Add as many sensors as you like per room. It only takes one to be open to pause heating.
  • Rooms disappear after reload: Use Save configuration before clicking Reload — unsaved changes are lost otherwise.
  • HACS update not showing: Make sure the version in manifest.json matches the GitHub release tag exactly.
